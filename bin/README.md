# Вспомогательные скрипты и утилиты (/opt/pmi/bin/)

В данной директории содержатся вспомогательные Bash и Python скрипты, предназначенные для настройки инфраструктуры, управления сетью и выполнения сервисных задач оркестратора NETSTORM (PMI).

⚠️ **Важно:** Все скрипты в этой папке предполагают запуск с правами суперпользователя (`root`), так как они взаимодействуют с системными параметрами (`sysctl`), сетевыми пространствами имен (`netns`) и системными сервисами.

---

## ⚙️ Системное окружение (.env)
Все скрипты оркестратора опираются на глобальные системные переменные, заданные в файле `/opt/pmi/.env`. 
Главный стартовый скрипт (`pmi_start.sh`) автоматически парсит этот файл при запуске, экспортируя `PMI_HOME`, `PYTHONPATH` и другие критичные параметры для Python-движка. 

*Если вы пишете новый bash-скрипт, всегда начинайте его с `source /opt/pmi/.env` или вызывайте его через `pmi_start.sh`.*

---

## Содержимое

### 1. `pmi_start.sh` (Главный Launcher)
Единая точка входа для запуска компонентов оркестратора NETSTORM.
**Основные функции:**
- Парсит файл `.env` и настраивает `PYTHONPATH`.
- Генерирует уникальный идентификатор сессии (`PMI_RUN_ID`), связывающий логи и отчеты.
- Проверяет наличие зависимостей (Python 3, PyYAML) и устанавливает их при отсутствии.
**Использование:**
- `./pmi_start.sh tui` — Запуск классической интерактивной консоли (TUI).
- `./pmi_start.sh web` — Запуск Web-дашборда в foreground-режиме (отладочный режим с uvicorn).
- `./pmi_start.sh service {start|stop|status}` — Управление systemd-демоном Web-оркестратора.

### 2. `backend.sh`
Скрипт для управления пулом Python-воркеров (тестовым бэкендом) внутри целевого сетевого пространства имен (по умолчанию `webserver`).
**Использование:** `./backend.sh {start|stop|status}`

### 3. `init_jmeter_ns.sh`
Скрипт создания и базовой настройки сетевого namespace для L7-генератора нагрузки (JMeter).
**Основные функции:**
- Создает изолированный `netns`.
- Переносит в него выделенный физический интерфейс.
- Настраивает базовый IP-адрес и шлюз по умолчанию.
- Автоматически вызывает скрипт `provision_ips.sh` для проливки пула фейковых адресов.
**Использование:** `./init_jmeter_ns.sh {start|stop}`

### 4. `nginx_switch.sh`
Скрипт быстрого переключения профилей конфигурации тестируемого веб-сервера (NGINX).
**Режимы:**
- `static`: конфигурация отдачи статики (оптимизировано для тестирования сетевой подсистемы).
- `backend`: проксирование трафика на Python-воркеры (для тестирования CPU и приложения).
- *Безопасность:* автоматически проверяет валидность конфигурации (`nginx -t`) перед выполнением `reload`.
**Использование:** `./nginx_switch.sh {static|backend|status}`

### 5. `provision_ips.sh`
Скрипт для пакетной (batch) проливки пула IP-адресов (IP Aliasing) на сетевой интерфейс внутри изолированного namespace генератора нагрузки (JMeter).
**Основные функции:**
- **Автоопределение интерфейса:** Находит нужный интерфейс внутри заданного `netns` (по умолчанию `vlan500-jmeter`), игнорируя loopback (`lo`).
- **Тюнинг ARP (Host OS):** Увеличивает лимиты сборщика мусора ARP (`net.ipv4.neigh.default.gc_thresh`), что критически важно при добавлении подсетей уровня `/16` (например, 60 000 адресов), чтобы ОС не отбрасывала записи.
- **Настройка rp_filter:** Переводит `rp_filter` в режим Loose mode (`2`) для поддержки асимметричной маршрутизации трафика при тестах.
- **Массовое применение (Batch Mode):** Считывает адреса из файла профиля (по умолчанию `/opt/pmi/profiles/jmeter/source_ips.csv`), формирует файл команд и применяет их через `ip -batch` за доли секунды.
**Использование:** `./provision_ips.sh`

## 🛠 Добавление новых скриптов

Если вы добавляете новый исполняемый скрипт в эту директорию:
1. Убедитесь, что файл имеет права на исполнение (`chmod +x script_name.sh`).
2. Обязательно загружайте переменные окружения в начале скрипта:
   ```bash
   if [ -f "/opt/pmi/.env" ]; then
       export $(grep -v '^#' /opt/pmi/.env | xargs)
   fi
   ```
3. Избегайте хардкода путей. Используйте переменную `$PMI_HOME` для навигации по структуре проекта.
4. Добавьте краткое описание нового инструмента в этот `README.md`.
