# Дашборды Grafana (PMI Dashboards)

В данной директории содержатся предварительно настроенные дашборды в формате JSON, предназначенные для импорта в систему визуализации **Grafana**. Они используются для мониторинга в реальном времени всех компонентов стенда (генераторов, цели и самой операционной системы) во время проведения ПМИ.

---

## 📊 Содержимое директории

### Основные дашборды

* **`control_center.json`** — **PMI: Control Panel (Mission Control)**
  Главный сводный дашборд. Показывает общее состояние системы: статус работы критических служб `systemd` (TRex, Nginx, VictoriaMetrics), потребление CPU/RAM в разрезе процессов генераторов, а также корреляционные графики (нагрузка L4/L7 vs влияние на процессор/NGINX).

* **`jmeter.json`** — **PMI: L7 Load (JMeter)**
  Специализированный дашборд для прикладного уровня. Отображает активные потоки (Active Threads), общую пропускную способность, RPS успешных и ошибочных запросов, время ответа (95th percentile) и объемы переданного/полученного трафика.

* **`trex.json`** — **PMI: L3 Load (TRex STL)**
  Специализированный дашборд для сетевого уровня L3/L4. Показывает TX/RX пакеты в секунду (PPS) и пропускную способность (BPS) с разбивкой по портам генератора, а также оценивает потери пакетов на лету (Est. Packet Loss).

* **`victim.json`** — **PMI: Victim Dashboard (Nginx)**
  Дашборд целевого сервиса (Жертвы). Опирается на `nginx-prometheus-exporter` и показывает: обработанный RPS, метрики соединений (Active, Idle, Reading, Writing, Waiting).

* **`host.json`** — **PMI: Host Monitoring**
  Классический системный мониторинг на базе `node_exporter`. Отслеживает утилизацию CPU, свободной памяти (в % и байтах), загрузку сети на интерфейсах хоста и состояние дисковой подсистемы.

## 🛠 Как импортировать дашборды

Для добавления дашбордов в вашу инсталляцию Grafana:

1. Откройте веб-интерфейс Grafana.
2. В левом меню перейдите в **Dashboards** -> **Import**.
3. Нажмите **Upload JSON file** и выберите нужный файл из этой папки (или просто скопируйте и вставьте содержимое файла в текстовое поле).
4. Нажмите **Load**.
5. При необходимости (если Grafana спросит) укажите ваш Data Source (по умолчанию в стенде используется **VictoriaMetrics** с uid `P4169E866C3094E38`).
6. Нажмите **Import**.

*Примечание: Все дашборды автоматически фильтруют метрики по времени и используют глобальные переменные для переключения между таргетами или интерфейсами (см. выпадающие списки вверху экранов).*
