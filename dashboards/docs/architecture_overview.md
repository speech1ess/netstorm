# 📚 Техническая документация NetStorm

Этот раздел содержит описание внутреннего устройства **NetStorm Orchestrator v1.0**. Документация предназначена для инженеров, занимающихся эксплуатацией стенда, доработкой модулей или интеграцией в CI/CD.

---

## 🗺 Карта разделов

Для удобства навигации архитектура разделена на три ключевых домена:

### ⚙️ [Устройство системы](SYSTEM_ARCHITECTURE.md)

* **Смысл:** Взаимодействие Python-движка с ОС и генераторами (TRex, JMeter). Логика и возможности оркестрации.
* **Темы:** Жизненный цикл теста (Data Flow), управление изоляцией трафика и сетевыми пространствами имен (`netns`), работа с процессами-акторами, профилями и параметрами нагрузки.
* **Цель:** Понять, как код превращается в нагрузку.

### 📝 [Конфигурационный слой](CONFIG_ARCHITECTURE.md)

* **Смысл:** "Data-Driven" подход. Управление через YAML без изменения кода.
* **Темы:** Иерархия `global.yaml` и `test_program.yaml`, наследование пресетов, передача динамических переменных (tunables) в профили.
* **Цель:** Создание сложных сценариев тестирования.

### 📊 [Отчетность и Мониторинг](REPORTS_ARCHITECTURE.md)

* **Смысл:** Сбор доказательной базы и анализ результатов.
* **Темы:** Изоляция сессий через `SESSION_ID`, Sidecar-мониторинг здоровья цели (Target Health), генерация агрегированных HTML-отчетов.
* **Цель:** Выявление false-positives. Оценка качества очистки трафика. Root Cause Analysis.
