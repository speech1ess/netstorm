## 📁 Структура директории

Директория логически разделена по используемым генераторам нагрузки (Load Generators):

### 1. `/trex/`
Содержит описания профилей нагрузки для генератора Cisco TRex (python, `*.py`). Разделяются на два типа API:
* **STL (Stateless):** Объемные атаки (Volumetric) и исчерпание состояния (State Exhaustion) без поддержания TCP-сессий (имена файлов соответствуют шаблону `stl_*.py`).
  * `stl_udp_mix.py` — Симуляция IMIX-трафика для утилизации полосы пропускания (L2/L3).
  * `stl_syn_flood.py` — TCP SYN флуд (отнимает ресурсы таблицы состояний фаерволов).
  * `stl_ack_flood.py` — TCP ACK флуд (нагрузка на Out-of-State обработку / Session Lookup).
  * `stl_rst_flood.py` — TCP RST флуд (Session Teardown Exhaustion).
  * `stl_icmp_flood.py` — Ковровая бомбардировка ICMP пакетами.
* **ASTF (Advanced Stateful):** Генерация честных TCP/UDP сессий для нагрузочного тестирования DPI и NGFW (имена файлов соответствуют шаблону `astf_*.py`).
  * `astf_http_cps.py` — Генерация создания новых соединений (CPS) по HTTP на высокой скорости.
  * `astf_mice_tcp_cps.py` — Генерация "мышиного" TCP-трафика (сверхкороткие сессии).

### 2. `/jmeter/` (L7 Прикладной уровень)
Содержит сценарии для Apache JMeter (XML, `.jmx`). Используются для атак прикладного уровня и создания модели эталонного "легитимного" фона (Baseline) для оценки качества очистки трафика (QoE). 

* **Профили (строгое именование с префиксом порядка):**
  * `00_baseline.jmx` — Эталонный профиль легитимных пользователей (поведенческая модель).
  * `01_get_flood.jmx` — Интенсивный HTTP GET-флуд.
  * `02_post_big.jmx` — Атака тяжелыми POST-запросами (утилизация канала и памяти).
  * ...
  * `11_ngfw_https_stress.jmx` — Комплексный стресс-тест криптопровайдера (TLS/SSL) на NGFW.

* **Датасеты параметризации (`.csv`):**
  * `legit_ips.csv` — Пул IP-адресов, имитирующих доверенных (легитимных) клиентов.
  * `attack_ips.csv` — Обширный пул IP-адресов для генерации распределенного ботнета (DDoS).
  * `uris_list.csv` — Динамический словарь URI для рандомизации запросов (Cache Bypass / Hash Collision).

## 📐 Архитектурные стандарты и правила разработки

Для обеспечения модульности и совместимости с оркестратором NetStorm, при написании новых профилей **обязательно** соблюдение следующих правил:

### Для TRex (Python):
1. **Именование классов:** Имя возвращаемого класса в профиле должно строго соответствовать индексу сценария в YAML-конфигурации (например, `class STLU1(object)` для UDP Flood, `class STLS1(object)` для SYN Flood).
2. **Никакого хардкода (Фоллбэки):** Целевые IP-адреса и пулы источников должны динамически приниматься из раннера через `kwargs`. В коде обязателен Fallback (значение по умолчанию) на случай ручного запуска профиля без оркестратора:
   ```python
   target_ip    = kwargs.get("dst_ip", "10.0.70.2")
   src_ip_start = kwargs.get("src_ip_start", "15.0.0.1")
   ```
3. **Абсолютный рандом:** Запрещено использование секвенсоров (`STLVmTupleGen`) для генерации IP или портов, так как это создает "лесенку" и легко блокируется базовой эвристикой Anti-DDoS. Используйте независимый рандом `STLVmFlowVar(op="random")`.
4. **Аппаратный Offload:** При написании TCP-флудов (STL) пересчет L4-контрольных сумм всегда должен передаваться на сетевую карту (NIC Hardware Offload) для экономии CPU:
   ```python
   STLVmFixChecksumHw(l3_offset="IP", l4_offset="TCP", l4_type=CTRexVmInsFixHwCs.L4_TYPE_TCP)
   ```

### Для JMeter (JMX):
1. **Отвязка от IP-адресов:** В `HTTP Request Defaults` адреса серверов (Target) должны передаваться через свойства (Properties): `${__P(target_host, 10.0.60.5)}`.
2. **Параметризация:** Привязка адресов к виртуальным сетевым интерфейсам (Local Address/Source IP) и чтение URI осуществляется строго через `CSV Data Set Config` из соответствующих файлов датасетов.

## 🛠 Как добавить новый тест?

Оркестратор работает по принципу **Data-Driven Execution**. Чтобы добавить новую атаку:

1. Создайте профиль по описанным выше стандартам (например, `stl_new_attack.py` или `12_new_l7.jmx`).
2. Положите файл в соответствующую директорию (`profiles/trex/` или `profiles/jmeter/`).
3. Откройте главный конфигурационный файл: `/opt/pmi/config/test_program.yaml`.
4. Зарегистрируйте новый профиль в секции `profiles` или напрямую в `scenarios`, передав ему необходимые переменные окружения и мультипликаторы.
5. Меню TUI и логика маршрутизации обновятся автоматически!
