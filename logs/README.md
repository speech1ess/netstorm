# Логи сессий и генераторов (PMI Logs)

Директория для хранения сырых "горячих" логов, которые пишутся в реальном времени во время работы оркестратора и генераторов трафика.

---

## 📁 Структура

Каждый запуск тестов создает уникальную подпапку `<SESSION_ID>` (например, `20260210_194850/`), куда складываются:

* **`pmi_session.log`** — Главный хронологический лог самого оркестратора. Содержит отладочную информацию о применении профилей, таймингах, запуске процессов и возможных ошибках.
* **`trex_<profile>_runX.log`** — STDOUT/STDERR процессы драйвера TRex. Сюда же раз в секунду сыплется статистика по PPS/BPS и потерям (Tx/Rx).
* **`jmeter_<profile>_runX.log`** — Индивидуальный лог процесса JMeter. Содержит дерево потоков и финальную сводку (summary), на основе которой парсер строит итоговые таблицы.
* **`target_metrics.csv`** — Таблица временных рядов (timestamp, CPU, RAM, Net), собираемая Sidecar-монитором целевого узла (только в режиме BACKEND).

---

## 🔄 Жизненный цикл
Данные в этой папке считаются "сырыми". Как только сценарий завершается (успешно или по Ctrl+C), скрипт `generate_run_summary.py` автоматически **копирует** нужные логи из этой папки в `../results/<SESSION_ID>/` и упаковывает их в ZIP-архив. 

*Примечание: Этот README-файл гарантирует присутствие директории `logs/` в Git.*
