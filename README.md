# NETSTORM Load Test Orchestrator v1

Программный комплекс для автоматизации и оркестрации нагрузочного тестирования. Позволяет запускать сложные сценарии, состоящие из атак L3/L4 (TRex STL) и L7 (JMeter, TRex ASTF), с управлением через Web-интерфейс или TUI, сбором метрик и автоматической генерацией отчетов.

## 📁 Структура проекта

* 📄 **`.env`** — Главный системный контракт (переменные окружения, пути, порты).
* 📂 **`bin/`** — Исполняемые bash-скрипты запуска, настройки окружения и netns.
* 📂 **`config/`** — Инфраструктурные (`global.yaml`) и тестовые конфигурации (`test_program.yaml`).
* 📂 **`dashboards/`** — Дашборды Grafana (JSON).
* 📂 **`docs/`** — Документация по архитектуре конфигов и отчетам.
* 📂 **`lib/`** — Ядро оркестратора (Движки тестов, модули, генераторы отчетов `lib/reporting/`).
* 📂 **`profiles/`** — Боевые скрипты атак (L3/L4 TRex) и сценарии (L7 JMeter).
* 📂 **`web/`** — Исходный код Web-оркестратора (FastAPI + Vue.js/Tailwind).
* 📂 **`logs/`** & **`results/`** — Директории сессионных логов и HTML-отчетов.

## 🚀 Быстрый старт

1. **Настройте окружение:** Проверьте базовые пути и порты в файле `/opt/pmi/.env`.
2. **Настройте инфраструктуру:** Убедитесь, что IP-адреса стенда и пути к бинарникам корректно указаны в `/opt/pmi/config/global.yaml`.
3. **Опишите логику тестов:** Создайте или отредактируйте профили в `/opt/pmi/config/test_program.yaml`.
4. **Запустите оркестратор:**
   ```bash
   cd /opt/pmi
   # Запуск Web-сервиса в фоне (production)
   sudo ./bin/pmi_start.sh service start
   
   # ИЛИ запуск консольного интерфейса (TUI)
   ./bin/pmi_start.sh tui
   ```
5. **Откройте Web-интерфейс:** Перейдите в браузере по адресу `http://<ip_оркестратора>:8000`.
6. **Выполните тест:** На вкладке "Сценарии" выберите нужный профиль нагрузки или запустите одиночный скрипт через "Профили (Explorer)".
7. **Просмотрите результаты:** Исторические HTML-отчеты доступны на порту `8080` (вкладка "Отчеты").

## 🏛️ Архитектура (Hybrid Control Plane)

Система использует гибридную архитектуру для обеспечения максимальной производительности генераторов:

* **Control Plane (Web/TUI):** Легковесный оркестратор на FastAPI, управляющий конфигурациями, жизненным циклом тестов и парсингом логов.
* **Data Plane (Load Generators):** TRex (L3/L4/L7) и JMeter (L7), работающие напрямую с ресурсами хоста (DPDK, HugePages, netns).
* **Target Server:** Nginx-заглушка, работающая в двух режимах: `STATIC` (для тестов сети) и `BACKEND` (для тестов CPU/приложения).
* **Monitoring:** VictoriaMetrics для сбора Time-Series данных и Grafana для визуализации метрик в реальном времени.

## ⚙️ Три слоя конфигурации

Оркестратор использует парадигму разделения среды и приложения:

1. **Слой ОС (`.env`):** Системные переменные (`PMI_HOME`, `PYTHONPATH`, порты). Загружаются автоматически стартовыми скриптами.
2. **Слой Инфраструктуры (`global.yaml`):** IP-адреса узлов, настройки сетевых интерфейсов, адреса метрик. Меняется редко, использует Jinja2 шаблонизатор.
3. **Слой Сценариев (`test_program.yaml`):** Динамическая логика тестирования. Связывает физические файлы (`.py` / `.jmx`) с алиасами и группирует их в сценарии с пресетами нагрузки (`low`, `medium`, `high`).

---

## 🌐 Web-Интерфейс (Дашборд)

Главный интерфейс управления системой (порт 8000). Предлагает 5 основных модулей:

* 🎯 **Сценарии:** Запуск боевых связок тестов, описанных в `test_program.yaml`. Поддержка пакетного запуска (Batch Run) и кастомных параметров нагрузки через модальные окна.
* 🗂️ **Профили (Explorer):** "Песочница" для Ad-Hoc запуска любых TRex и JMeter скриптов напрямую с диска, без необходимости прописывать их в конфигурации.
* 🛠 **Диагностика:** Хелсчеки сети, сервисов и переключение режимов Target-сервера.
* ⚙️ **Конфиги:** Выбор активного YAML-файла сценариев на лету.
* 🗄️ **Логи и База:** Просмотр свободного места и утилиты ротации/очистки старых отчетов и логов.

## 📈 Отчеты и Анализ

После каждого успешного запуска автоматически запускается фоновый процесс (`lib/reporting/generate_run_summary.py`), который формирует результаты:

* **Главный индекс:** Календарь всех запусков со статусами (успех/ошибка) и ссылками на индивидуальные страницы.
* **Отчет сессии:** Детальный HTML-отчет, включающий сводные таблицы TRex, графики JMeter и встроенный просмотрщик логов для дебага.
* **Grafana Dashboards:** Ссылки на преднастроенные дашборды для детального покадрового анализа атаки.

## 🛠️ Добавление новых тестов

1. **Создайте файл профиля:** Положите ваш `.py` (TRex ASTF/STL) или `.jmx` (JMeter) в `/opt/pmi/profiles/`. (Скрипт уже доступен для запуска во вкладке "Профили").
2. **Привяжите к сценарию (опционально):** Откройте активный `test_program.yaml`, добавьте профиль в секцию `profiles` и создайте для него сценарий в `scenarios`.
3. **Запускайте!** Оркестратор динамически перестроит меню и Web-интерфейс.
